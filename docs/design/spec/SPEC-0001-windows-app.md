# インフォグラフィック プレゼンテーションツール
## Windows アプリケーション仕様書 v2.0

> v1.0（Electron/React 計画）から v2.0（WPF/.NET 9 実装）へ全面改訂。  
> 技術選定の変更経緯は `.udr/records/` 参照。

---

## 1. プロダクト概要

### 1-1. コンセプト

大判のインフォグラフィック画像・PDF・設計図をそのまま使い、パン／ズームのシナリオ（キーフレーム列）を組み立ててプレゼンテーションとして再生する Windows デスクトップアプリ。

### 1-2. ターゲットユーザー

- 社内プレゼンにインフォグラフィックを使う営業・企画担当者
- A3/A0 サイズの全体図から各論を掘り下げて説明したいエンジニア・PjM
- draw.io / PDF / PNG 形式の図をそのままプレゼンしたいユーザー
- Prezi を使いたいが社外 SaaS 導入が難しい組織

### 1-3. 提供価値

| 課題 | 本アプリの解決策 |
|------|----------------|
| Prezi は SaaS／要ログイン | ローカル完結・オフライン対応 |
| PowerPoint に画像を貼ると解像度が落ちる | 原寸画像のままズームして高精細を維持 |
| シナリオを作るのが難しい | ドラッグ＋スクロールで直感的に登録 |
| 複数の画像・ページを切り替えたい | 複数画像管理とプレゼン中の画像切替に対応 |

## 2. 技術スタック

### 2-1. 採用技術

| レイヤ | 技術 | 選定理由 |
|--------|------|---------|
| フレームワーク | **.NET 9 / WPF / C#** | Windows ネイティブ、高品質レンダリング、デプロイが単純 |
| SVG レンダリング | **SharpVectors.Reloaded 1.8.4** | WPF 向け SVG 描画ライブラリ |
| PDF ラスタライズ | **PDFtoImage 5.0.0** | PDF を画像化（200dpi） |
| draw.io 変換 | **draw.io Desktop CLI**（外部ツール） | ネイティブ XML を PDF 経由で取り込み |
| シリアライズ | **System.Text.Json 9.0.0** | .przip 内 JSON の読み書き |
| インストーラ | **WiX v5 (MSI)** | ファイル関連付け・ファイアウォール規則登録 |
| ビルド | **dotnet publish** (単一ファイル・自己完結型) | .NET ランタイム同梱で配布が容易 |

### 2-2. 代替案（参考）

| 技術 | 特徴 | 不採用理由 |
|------|------|----------|
| Electron + React | Web 資産の再利用が容易 | バイナリが大きく（150MB+）、WPF より描画品質が劣る。当初計画したが WPF に変更 |
| Tauri (Rust+WebView2) | バイナリが小さい | Rust ビルド環境の整備コスト |

## 3. 機能要件

### 3-1. 画像管理（F-01）

- 対応形式: **JPG / PNG / BMP / GIF / WebP / SVG / PDF / draw.io（`.drawio` / `.dio`）**
- ファイルダイアログまたは円形ドロップゾーンへのドラッグ＆ドロップで読み込み
- PDF は全ページを個別画像として展開（`<ファイル名>_p01.png` 形式、200dpi）
- draw.io ネイティブ形式は draw.io Desktop CLI を使って PDF 変換後に取り込み
  - draw.io Desktop の自動検出パス:
    - `%LOCALAPPDATA%\Programs\draw.io\draw.io.exe`
    - `%ProgramFiles%\draw.io\draw.io.exe`
    - 環境変数 `DRAWIO_PATH`
  - 「Editable SVG/PNG エクスポート」形式はツールなしで直接処理
- **複数画像対応**: 1プロジェクトに複数の画像・ページを格納可能
  - 画像リストでドラッグ＆ドロップ並べ替え
  - `Ctrl+Tab` / `Ctrl+Shift+Tab` で次/前の画像へ切替
- 最大推奨解像度: **8192×8192 px**（超過時は警告ダイアログを表示）
- 「画像差し替え」: 現在選択中の画像のみ別ファイルに置換（紐付くシーンは保持）

### 3-2. キャンバス操作（F-02）

- **パン**: 左クリック＋ドラッグ
- **ズーム**: マウスホイール（カーソル位置中心）、`Shift+ホイール` で 10% 刻み
- **ズームキー**: `+` / `-`（2% 刻み）、`F` でフィット表示
- 画像表示モード: 「画面にフィット」（既定）/ 「原寸表示（100%）」

### 3-3. シナリオ編集（F-03）

サイドバーの各シーンカードから操作:

- シーンの **追加**（現在ビューをキャプチャ）/ **削除** / **複製** / **名前編集**
- **↑↓ ボタンまたはドラッグ＆ドロップ**で並べ替え
- **↻ 現在ビューで上書き**: 開いているビューでシーンを更新
- **👁 プレビュー**: シーンへアニメーション遷移して確認
- **遷移時間スライダー**: 0.2 秒 〜 4.0 秒
- **画像切替効果**: 無し / フェード / クロスフェード
- **絞り込み検索**: シーン名でフィルタ（🔍）

### 3-4. プレゼン再生（F-04）

- **全画面モード**（F11）
- ナビゲーション:

  | キー | 動作 |
  |------|------|
  | → / Space / PageDown | 次のシーン |
  | ← / PageUp | 前のシーン |
  | Shift+→ / Shift+Space | 次の画像 |
  | Shift+← | 前の画像 |
  | Home / End | 最初 / 最後のシーン |
  | 1〜9 | シーン番号でジャンプ |
  | Shift+1〜9 | 画像番号でジャンプ |
  | F5 | 最初から再生 |
  | B | ブラックアウト（任意キーで解除） |
  | T | ツールバー表示/非表示 |
  | H / ? | キーボードショートカット一覧 |
  | Esc | 描画OFF → ガイド閉 → プレゼン終了（段階的） |

- **表示スケールモード**: 「横幅に合わせる」/「縦幅に合わせる」（ツールバーで選択）

### 3-5. 描画（アノテーション）（F-05）

- プレゼン中に `D` キーで描画モードに切替
- 色（赤/黄/緑/青/白）と太さをツールバーで選択
- `E` キーで消しゴムモード、`C` キーでこの画像の描画をクリア
- `Ctrl+S` またはツールバーの「S」で描画を PNG として書き出し
- 描画データは `.przip` には保存されない（一時的なアノテーション）

### 3-6. ファイル操作（F-06）

- 独自形式 **`.przip`**（画像＋シナリオを ZIP 圧縮）で保存

  | ショートカット | 動作 |
  |---|---|
  | Ctrl+N | 新規作成 |
  | Ctrl+O | ファイルを開く |
  | Ctrl+S | 上書き保存 |
  | Ctrl+Shift+S | 名前を付けて保存 |
  | Ctrl+Z / Ctrl+Y | 元に戻す / やり直し |

- 最近使ったファイル（MRU）: 「ファイル」メニューから開く
- **起動時モード**（`.przip` に保存）: エディタ / プレゼン
  - `.przip` ダブルクリック時は保存済みモードで起動
  - Shift 押下で起動時モードをエディタに上書き

### 3-7. エクスポート（F-07）

- 「ファイル」→「エクスポート」から:
  - **全シーンを PNG 連番で書き出し**: 各シーンを画像ファイル化
  - **描画を PNG で保存**: 描画がある画像を PNG で書き出し

### 3-8. コマンドライン起動（F-08）

```
InfoDive.exe [オプション] [ファイル ...]
```

| オプション | 動作 |
|---|---|
| `-e` / `--editor` | エディタモードで開く |
| `-p` / `--present` | プレゼンモードで開く |
| (なし) | `.przip` に保存された起動時モードに従う |

- `.przip` 1 ファイル指定 → プロジェクトを開く
- 画像 / PDF / draw.io ファイルを複数指定 → すべて読み込み
- `.przip` と他ファイルが混在 → `.przip` が優先

## 9. セキュリティ方針

- **ファイアウォール規則**: MSI インストール時に InfoDive の送信接続をブロック（オフライン動作の保証）。アンインストール時に自動削除。
- **ファイルアクセス**: ユーザーが明示的に開いたファイルのみ読み書き。`app.getPath` 相当は `%APPDATA%\InfoDive\` 以下を使用。
- **外部ツール**: draw.io Desktop の自動検出パスは固定リスト（相対パスは使用しない）。環境変数 `DRAWIO_PATH` は絶対パスのみ受け付ける。
- **パッケージ**: NuGet 依存パッケージは週次 Dependabot + CI の脆弱性チェック（`dotnet list package --vulnerable`）で継続監査。
